# Medigent — Pitch deck outline (draft)

## 0) One-liner
AI-powered medical pre-consultation that reduces clinician time and increases patient throughput while maintaining safety.

## 1) Title
Medigent — AI Medical Pre‑Consultation (USA focus)

## 2) Problem
- Clinics waste clinician time on repetitive intake + incomplete patient histories.
- Patients arrive unprepared; data quality is inconsistent.
- Clinics need throughput + revenue per clinician hour.

## 3) Solution
- Patient-facing pre-consultation flow (chat/form) that collects symptoms, history, meds, red flags.
- Outputs structured summary for clinician + EHR-ready fields.
- Safety: escalation rules / disclaimers / handoff.

## 4) Product
- UX screenshots / demo flow
- What is automated vs what remains human
- Integration points (EHR, scheduling, telehealth)

## 5) Why now
- Clinics pressured on margins and staffing.
- Patients used to digital intake.
- LLM + workflow tooling now viable.

## 6) Market
- ICP (who pays): specialty clinics / urgent care / telehealth / primary care groups
- TAM/SAM/SOM (keep credible)

## 7) Business model
- Pricing per clinic / per provider / per consultation
- Implementation + integration fees (if any)

## 8) Traction
- MVP built
- Users / pilots / LOIs / waitlist / partnerships
- Key metrics (conversion, minutes saved, NPS)

## 9) Go-to-market (USA)
- Wedge (1 specialty)
- Channels: outbound to clinic managers + partnerships (EHR/telehealth) + founders’ network
- Sales cycle assumptions

## 10) Competition
- Traditional intake tools
- AI symptom checkers
- Differentiation: clinician workflow + structured output + integration + safety

## 11) Defensibility
- Workflow + data + integrations
- Clinical + regulatory posture
- Distribution partnerships

## 12) Team
- 3 founders: roles
- What’s missing: USA GTM / fundraising cofounder

## 13) Ask
- What you’re raising (amount + runway + milestones)
- Or: what you want from this VC conversation (advice, intro, pre-seed interest)

## 14) Appendix
- Demo video link
- Pilot case study
- Security/compliance overview (HIPAA posture, SOC2 plan)
