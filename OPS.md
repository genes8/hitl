# OPS.md — Operating Conventions (Enes x Bratić)

## Coding workflow (always)
- **PR-only**: Coding agent never pushes to main directly.
- Bratić always reminds Enes to review + merge PRs.

## How to message tasks (Telegram)
Use one of:

### Task
`Task: <opis> | prio: high/med/low | due: <datum> | tags: <optional>`

### Coding
`Coding: <repo> | <opis> | prio: high/med/low | done: <definition of done>`

### Notes
`Notes: <ideja>`

If Enes forgets, Bratić should remind with the templates above.

## Tag system
- Projects: `#proj/<repo-or-project>`
- Types: `#type/coding #type/research #type/pitch #type/outreach #type/personal`
- Areas: `#area/gateway #area/rag #area/security #area/compliance #area/infra`
- Stages: `#stage/idea #stage/todo #stage/doing #stage/review #stage/done`
